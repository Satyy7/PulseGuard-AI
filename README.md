# 🚀 PulseGuard AI

### Real-Time Anomaly Detection & AI Incident Investigation Platform

PulseGuard AI is a real-time observability and incident intelligence platform designed to detect anomalies in distributed microservices and automatically generate AI-powered root cause analysis reports.

The system simulates a microservice infrastructure, streams telemetry through Kafka, detects anomalies using machine learning, aggregates incidents, and provides AI-driven investigation reports through an interactive dashboard.

PulseGuard demonstrates how modern observability platforms like Datadog, Dynatrace, and New Relic internally process system telemetry to help engineers diagnose production issues faster.

---

# System Overview

PulseGuard continuously monitors system metrics generated by multiple microservices. These metrics are streamed through Kafka, analyzed by a machine learning anomaly detection engine, aggregated into incidents, and stored in PostgreSQL.

An AI investigation engine then analyzes incidents and generates structured root cause analysis reports that engineers can view in a dashboard.

## Main capabilities

- Real-time metrics streaming
- Machine learning anomaly detection
- Incident aggregation and storage
- AI-powered root cause analysis
- Observability dashboard for monitoring and investigation

# Core System Components

## Metric Generator

Simulates production microservices by continuously generating telemetry data such as:

- CPU usage
- Memory usage
- Request latency
- Error rates

These metrics are published to the Kafka topic **system-metrics**.

---

## Kafka Streaming Layer

Kafka acts as the real-time event backbone of the system.

### Topics used

- system-metrics → Raw infrastructure metrics
- anomaly-events → Detected anomalies

---

## Anomaly Detection Engine

Consumes metrics from Kafka and detects anomalies using a hybrid approach combining machine learning and statistical methods.

### Detection techniques

- Isolation Forest
- Z-score anomaly detection

When abnormal behavior is detected, anomaly events are published to Kafka.

---

## Incident Aggregator

The aggregator groups multiple anomalies occurring within a time window and determines whether they represent a real system incident.

If the anomaly threshold is reached, an incident is created and stored in PostgreSQL.

Incident severity is determined based on the triggering metric.

Example thresholds:

Latency > 500 ms → HIGH  
CPU > 90% → HIGH  
Error Rate > 5 → HIGH  
Memory > 90% → MEDIUM

---

## FastAPI Investigation API

The API layer exposes endpoints to retrieve:

- system metrics
- logs
- incidents
- dependency traces

It also triggers the AI investigation engine.

---

## AI Root Cause Analysis Engine

PulseGuard integrates with a large language model through the Groq API to automatically investigate incidents.

The AI agent analyzes:

- incident timeline
- service metrics
- logs
- dependencies

### AI Report Structure

Service  
Incident Timeline  
Root Cause  
Impact  
Recommended Fix

---

## Streamlit Observability Dashboard

The dashboard provides an interface for engineers to monitor and investigate system incidents.

### Dashboard Features

- Live incident monitoring
- Incident history
- Service metrics visualization
- Logs and dependency tracing
- AI investigation interface

# Installation

## Clone the repository

git clone https://github.com/satyy7/pulseguard-ai.git

cd pulseguard-ai

---

## Start infrastructure

docker-compose up -d

This starts the following services:

Kafka → port 9092  
PostgreSQL → port 5432  
Redis → port 6379

---

## Install dependencies

pip install -r requirements.txt

Example dependencies:

fastapi  
uvicorn  
kafka-python  
psycopg2-binary  
streamlit  
pandas  
scikit-learn  
joblib  
groq  
python-dotenv

---

# Running the System

Run each component in a separate terminal.

---

Start Metric Generator

python metric_generator.py

This service simulates real-time system metrics.

---

Start Anomaly Detection

python anomaly_consumer.py

Consumes Kafka metrics and detects anomalies.

---

Start Incident Aggregator

python incident_aggregator.py

Aggregates anomalies and stores incidents in PostgreSQL.

---

Start FastAPI Server

uvicorn main:app --reload

API will be available at:

http://127.0.0.1:8000

---

Start Dashboard

streamlit run dashboard.py

Dashboard will be available at:

http://localhost:8501
